Version 0.1.4 — 2007-12-22

  • License: RegexPal is now released under the LGPL. There is also a Google Code project where you are welcome to participate.
  • Enhancement: Trailing range-hyphens in unclosed character classes are not highlighted as errors.
  • Fix: A leading, literal newline in the regex field doesn't cause highlighting misalignment (IE only).
  • Fix: Control characters can be specified in lowercase (i.e. \ca is equivalent to \cA).

Version 0.1.3 — 2007-11-10

  • License: RegexPal is now released under the Creative Commons Attribution–Share Alike License.

Version 0.1.2 — 2007-10-18

  • Enhancement: A new link at the bottom of the page allows easily generating a permalink to the current regex, input, and flags.
  • Fix: Some valid interval quantifiers such as {20,100} were highlighted as errors (v0.1.1 only).

Version 0.1.1 — 2007-08-30

  • New feature: The regex, test data, and flags used when the page first loads can be specified in the query string. Examples:
    • Zip codes
    • HTML tags, case insenstive
    • Innermost tables in HTML source code
  • Fix: Interval quantifiers in reverse numeric order such as {3,1} are highlighted as errors.
  • Fix: Interval quantifiers with values greater than 65,535 such as {0,65536} are highlighted as errors to mimic Firefox's handling.

  • First release.
